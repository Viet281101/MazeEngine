# Supabase Backend (Local + Future Production)

This folder contains database schema and local Supabase setup artifacts.

## Prerequisites

- Docker Desktop (or Docker Engine)
- Supabase CLI
- Docker socket access for your current shell session

## First-time setup

From the repository root:

```bash
npm install
npm run supabase:init
npm run supabase:start
npm run supabase:reset
npm run supabase:status
```

Then:

1. copy `frontend/.env.example` to `frontend/.env`
2. copy the local Supabase URL and anon key from `npm run supabase:status`
3. start the full stack with `npm run dev`

## Useful commands

- Start: `npm run supabase:start`
- Stop: `npm run supabase:stop`
- Status: `npm run supabase:status`
- Reset DB: `npm run supabase:reset`

## Folder Contents

- `migrations/`: SQL migration history
- `seed.sql`: optional local seed data
- `config.toml`: Supabase local config generated by `supabase init`

## Current schema

- `public.mazes`: user-owned maze records (`maze_data` stored as `jsonb`)
- RLS enabled, each authenticated user can only access their own rows.

## Troubleshooting

- If `supabase start` fails with Docker permission errors, your shell likely does not have access to the `docker` group yet.
- After adding your user to the `docker` group, fully restart the shell session or log out and back in before retrying.
